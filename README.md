# 时刻声场 Audio Timer

一个纯前端的定时音频播放器，支持本地音频上传、播放队列、音频可视化和定时播放。项目不需要后端服务，已经发布到 GitHub Pages。

## 在线访问

[https://boredlittlenan.github.io/Audio-player/](https://boredlittlenan.github.io/Audio-player/)

## 主要功能

- 添加本地音频文件，支持 MP3、WAV、FLAC、M4A、AAC、OGG 等常见格式
- 拖拽音频到播放器快速加入歌单
- 播放、暂停、上一首、下一首、进度拖动、音量调节
- 支持 0.75x、1x、1.25x、1.5x、2x 倍速播放
- 音频频谱可视化和唱片动效
- 支持指定日期时间播放
- 支持倒计时播放，并提供 5、15、30、60 分钟快捷按钮
- 支持取消定时任务
- 自动保存歌单、当前曲目、播放进度、音量和倍速

## 数据保存方式

播放器完全运行在浏览器本地，不会把用户音频上传到服务器。

- 音频文件保存在浏览器的 IndexedDB 中
- 当前曲目、播放进度、音量、倍速和定时面板输入保存在 localStorage 中
- 用户关闭并重新打开同一个浏览器后，可以恢复上次的歌单和播放位置

注意：浏览器关闭后，页面里的定时器不会继续运行。也就是说，定时播放需要页面保持打开；重新打开页面后，上次的播放数据会恢复。

## 本地使用

直接打开 `index.html` 即可使用。

也可以启动一个简单的静态服务器：

```bash
npx serve .
```

然后在浏览器打开服务提示的本地地址。

## 技术说明

- 无框架、无构建步骤，使用原生 HTML、CSS 和 JavaScript
- 使用 `<audio>` 播放本地音频
- 使用 Canvas 绘制音频可视化
- 使用 Web Audio API 获取频谱数据
- 使用 IndexedDB 保存音频文件
- 使用 localStorage 保存轻量播放状态

## 浏览器兼容

建议使用最新版 Chrome、Edge、Firefox 或 Safari。部分浏览器会限制自动播放，如果定时触发时被拦截，页面会提示用户手动点击播放。
